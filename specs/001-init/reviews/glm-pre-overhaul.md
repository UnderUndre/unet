### Непроверенные допущения

- [ ] **AmneziaWG Docker-образ хорошо работает с программным клиентом `wgctrl`** — опровергается: разверните AmneziaWG в Docker, сгенерируйте конфигурацию клиента, затем подключитесь с помощью клиента `wgctrl` на Go на Linux, macOS и Windows. AmneziaWG добавляет обфускацию (SaltyDice, заголовки junk), которая может быть несовместима со стандартным клиентом WireGuard. Если `wgctrl` не может установить обфусцированные параметры AmneziaWG, туннель не установится.
- [ ] **`wgctrl` работает на Windows без WireGuard Windows service** — опровергается: попробуйте использовать `wgctrl` на свежей машине с Windows 10/11 без установленного WireGuard. На Windows `wgctrl` делегирует работу встроенному сервису WireGuard (tunnel.dll). Если сервис не установлен, он молча завершается сбоем или требует wireguard-go в userspace, что не задокументировано в плане.
- [ ] **Caddy REST API поддерживает токен-аутентификацию** — опровергается: проверьте документацию Caddy 2.x admin API. Стандартный Caddy поддерживает аутентификацию по IP-привязке, но не имеет встроенной поддержки Bearer-токенов без пользовательского middleware. Спецификация (`FR-008`) предполагает, что аутентификация по токену является опцией. Проверьте, существует ли плагин или нативная поддержка.
- [ ] **`docker-compose up -d` работает на свежей Ubuntu VPS** — опровергается: попробуйте на минимальном VPS-образе Ubuntu. Современные образы поставляются с Docker, но не с `docker-compose` (v1) — вместо этого это `docker compose` (v2 плагин). Команда provisioning должна использовать правильную.
- [ ] **Пользователь имеет доступные SSH-ключи, а не аутентификацию по паролю** — опровергается: протестируйте `ssh.go` с помощью аутентификации по паролю SSH. `data-model.md` показывает только `privateKeyPath` — нет поля пароля или опции для учетных данных на основе пароля. Многие пользователи VPS используют пароли по умолчанию.
- [ ] **30MB-ный двоичный файл достижим с помощью embedded React + Go + wgctrl** — опровергается: соберите минимальный двоичный файл с помощью `go:embed`, обслуживающий приложение React + `wgctrl`, и проверьте размер. React-комплект (gzip) может быть 500KB-2MB, но статика Vite + binary Go + зависимости могут превышать 30MB, особенно на Windows.

### Сбои

- [ ] **Разрыв соединения SSH во время provisioning (T009)** — что происходит сейчас: скрипт `docker-compose` прерывается на середине, возможно, оставляя полу-инициализированные контейнеры; что должно происходить: неинициализированное повторное выполнение скрипта provisioning (идемпотентность). Нигде не упоминается идемпотентность.
- [ ] **DELETE маршрута Caddy по индексу (contracts/caddy-api.md:38-41)** — что происходит сейчас: если два порта выставлены одновременно и один удаляется, индекс смещается, вызывая удаление неправильного маршрута; что должно происходить: атомарное совпадение и удаление, или идемификация маршрута по host, а не по позиции в массиве.
- [ ] **Несколько выставленных портов, пытающихся использовать один и тот же поддомен** — что происходит сейчас: нет описанной проверки; что должно происходить: отклонение с ошибкой конфликта.
- [ ] **Демон запускается, но VPN-туннель падает через 24 часа (повторный ключ WireGuard)** — что происходит сейчас: нет обработки повторного ключа в tunnel/manager; что должно происходить: повторное согласование ключей WireGuard автоматически или предупреждение.
- [ ] **Certbot/Let's Encrypt сбой при исчерпании лимита (spec.md:70)** — что происходит сейчас: краш при provisioning; что должно происходить: изящная деградация с понятным сообщением об ошибке и возвратом.
- [ ] **Состояние конфигурации (`config.json`) повреждено при сбое в середине записи** — что происходит сейчас: вероятно, усеченный JSON, демон не запускается при следующем запуске; что должно происходить: атомарная запись (запись во временный файл + переименование).

### Пограничные случаи

- [ ] **Null/Empty (Пустое/Нулевое)**: `config.json` не существует при первом запуске — должен создаваться с дефолтами. SSH `privateKeyPath` указывает на несуществующий файл. `exposedPorts` массив пуст. DNS `baseDomain` пуст в ручном режиме.
- [ ] **Boundary (Границы)**: 65535 выставленных портов (не реалистично, но нет задокументированных ограничений). Поддомены с метками >63 символов (ограничение DNS). Индексы маршрутов Caddy: первый (0) и последний маршрут.
- [ ] **Concurrent (Параллельные)**: Пользователь нажимает "Expose Port" дважды быстро — два идентичных маршрута Caddy создаются. REST API демона не имеет параллелизма. Пользователь запускает provisioning, пока туннель активен.
- [ ] **Stale (Устаревшие)**: `config.json` содержит `"status": "connected"`, но демон перезапускается после сбоя — туннель фактически не работает. `isProvisioned: true`, но Docker-контейнеры VPS были очищены вручную. Токен Caddy в конфигурации был отозван.
- [ ] **Malicious (Вредоносные)**: Инъекция SSH-команд через VPS-хост (пользователь вводит `"; rm -rf /` в поле хоста). Поддомен содержит `{` или специальные символы — инъекция JSON в payload Caddy. Порт设置为 `0` или отрицательный. Поле поддомена设置为 `../../../etc/passwd` (инъекция пути, если используется для генерации файлов).
- [ ] **Platform (Платформа)**: Пути Windows (`C:\Users\...`) для `privateKeyPath` — `wgctrl` ожидает формат, совместимый с ОС. Часовой пояс: срок действия сертификата Let's Encrypt — это UTC, но отображение пользователю может быть локальным. Локаль: ошибки SSH на локализованных Linux VPS (французский Ubuntu). Unicode: поддомены с нелатинскими символами. Windows: `sudo` не существует — quickstart.md показывает `sudo go run` — нужен раздел `Run As Administrator`.

### Отклоненные альтернативы

- [ ] **Tailscale/Headscale в качестве бэкенда** — отклонено, потому что: не оценено. AmneziaWG был выбран без документирования причин, по которым Tailscale/Headscale (который решает NAT traversal, key management, DERP relays и coordination server) был менее подходящим. Учитывая, что спецификация описывает "альтернативу Tailscale", это вызывает вопросы.
- [ ] **База данных на основе файлов (SQLite/BoltDB) для конфигурации** — отклонено, потому что: не оценено. `config.json` не масштабируется, если выставленные порты часто добавляются/удаляются (условие гонки при записи). BoltDB/SQLite решает проблему атомарности.
- [ ] **Токен-аутентификация демона (безопасность localhost UI)** — отклонено, потому что: не оценено. Любой локальный процесс может достучаться до `localhost:8080` и выполнить SSH-команды на вашем VPS или изменить конфигурацию. Нет куки-аутентификации для UI или демона.
- [ ] **Локальный API демона (contracts/)** — отсутствует контракт на REST API демона. Caddy API задокументирован, но нет спецификации для `localhost:8080/api/*`, которую вызывает frontend.
- [ ] **Альтернатива WireGuard-userspace (wireguard-go)** — отклонено, потому что: в исследовании упоминается, но нет плана использования/запасного варианта. На macOS `wgctrl` делегирует работу `wireguard-go` — это зависимость, которая должна быть либо встроена, либо предустановлена.

### Рекомендуемые действия

- **Приоритет 1 (блокирующие):**
  1. **Проверить совместимость AmneziaWG + `wgctrl`** — AmneziaWG использует проприетарные расширения handshake (Junk packets, Transport header). Стандартный `wgctrl` их не устанавливает. Если они не совпадают, туннель никогда не подключится. Протестируйте, прежде чем писать код. Если несовместимо: либо (a) используйте стандартный WireGuard и потеряйте обход DPI, либо (b) вызовите `awg-quick` вместо `wgctrl`, либо (c) используйте fork/patch `wgctrl` с поддержкой Amnezia.
  2. **Документировать контракт демона REST API** — frontend не может быть разработан без него. Создайте `contracts/daemon-api.md`, перечисляющий все `localhost:8080/api/*` эндпоинты с методами, payload-ами и кодами ответов.
  3. **Исправить уязвимость удаления маршрута Caddy (инъекция индекса)** — `DELETE /routes/[index]` — это классическая race condition. Используйте идемпотентное совпадение по host, а не позиционный индекс.

- **Приоритет 2 (перед релизом):**
  4. **Добавить поддержку аутентификации по паролю SSH** — `data-model.md` имеет только `privateKeyPath`. Многие пользователи облачных VPS начинают с паролей. Добавьте поле `password` (опционально, либо то, либо другое).
  5. **Использовать атомарные записи в `config.json`** — записывать во временный файл, затем переименовывать. Это бесплатно на POSIX и возможно на Windows.
  6. **Добавить контракт API демона, защищающий от concurrent exposure** — мьютекс или идемпотентный ключ для POST expose-port.
  7. **Уточнить требования Windows для `wgctrl`** — либо bundled WireGuard, либо опция wireguard-go. Задокументируйте в quickstart.md.
  8. **Использовать `docker compose` (v2), а не `docker-compose` (v1)** в скриптах provisioning — v1 устарел.

- **Приоритет 3 (технический долг / только пометка):**
  9. **Рассмотреть BoltDB/SQLite для хранения конфигурации** — JSON-файл подходит для MVP, но会在 условиях гонки при concurrent writes.
  10. **Добавить аутентификацию демона localhost UI** — в настоящее время любой локальный процесс имеет полный контроль. Приемлемо для MVP, но документируйте риск.
  11. **Документировать причину отклонения Tailscale/Headscale** — спецификация говорит "альтернатива Tailscale", но не объясняет, почему не Headscale.
  12. **Добавить идемпотентность provisioning** — скрипты должны быть безопасными для повторного запуска.
