## Entities

### WizardState

Persisted partial state of the first-run wizard. One active session maximum.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `session_id` | UUID (string) | yes | Unique session identifier. Generated on `POST /v1/wizard/sessions`. |
| `current_step` | enum | yes | Current wizard step. Values: `welcome`, `ssh`, `preflight`, `domain_mode`, `domain_check`, `cloudflare`, `commit`, `success` |
| `status` | enum | yes | Session status. Values: `in_progress`, `committed`, `abandoned` |
| `inputs` | object | yes | Accumulated step inputs (see below). |
| `preflight_result` | PreflightResult? | no | Populated after preflight step completes. |
| `domain_check_result` | DomainCheckResult? | no | Populated after domain check step completes. |
| `progress_pct` | integer | yes | Percentage complete (0-100). Computed from current_step. |
| `started_at` | ISO-8601 | yes | Session creation timestamp. |
| `last_saved_at` | ISO-8601 | yes | Last state persistence timestamp. |
| `committed_at` | ISO-8601? | no | Set when commit succeeds. Triggers state file deletion. |

**`inputs` structure**:

```json
{
  "ssh": {
    "host": "1.2.3.4",
    "port": 22,
    "user": "root",
    "auth_type": "key",
    "key_path": "~/.ssh/id_ed25519"
  },
  "domain_mode": "byo",
  "domain": "example.com",
  "cloudflare_token": "op:cf-...",  // redacted in logs, stored in state file
  "nipio_enabled": false,
  "first_peer_name": "phone",
  "first_port_expose": {
    "local_port": 3000,
    "subdomain": "app"
  }
}
```

**Validation rules**:
- `session_id`: UUID v4 format, generated by server.
- `current_step`: must follow step order (no skipping). Exception: back-navigation resets to earlier step.
- `inputs.ssh`: validated on ssh step submission (TCP connect + SSH auth + sudo docker ps).
- `inputs.domain_mode`: one of `byo` or `nipio`.
- `inputs.cloudflare_token`: validated only if `domain_mode == "byo"` and Cloudflare detected.
- `inputs.first_peer_name`: matches `^[a-zA-Z0-9_-]{1,64}$` (same as 002's CreatePeerRequest).

**Persistence**:
- File: `~/.unet/wizard-state.json`
- Mode: 0600
- Atomic write: temp file + rename (same pattern as 001-init config writes)
- Deleted on successful commit (`committed_at` set → file removed)
- Survives daemon restart (file-based, not in-memory)

### PreflightResult

VPS compatibility check result. Generated by `internal/wizard/preflight.Run()`.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `target_host` | string | yes | VPS IP or hostname checked. |
| `checked_at` | ISO-8601 | yes | When preflight ran. |
| `distro` | string | yes | Detected OS ID (e.g., `ubuntu`, `debian`). |
| `distro_version` | string | yes | Detected VERSION_ID (e.g., `22.04`, `12`). |
| `arch` | string | yes | CPU architecture (`x86_64`, `aarch64`). |
| `disk_free_gb` | float | yes | Available disk space on `/` in GB. |
| `ram_mb` | integer | yes | Total RAM in MB. |
| `has_sudo` | boolean | yes | Passwordless sudo available. |
| `has_docker` | boolean | yes | Docker binary present. |
| `docker_running` | boolean | yes | Docker daemon responding. |
| `port_443_free` | boolean | yes | TCP port 443 not bound by non-unet process. |
| `port_80_free` | boolean | yes | TCP port 80 not bound (needed for HTTP-01 TLS). |
| `port_wg_free` | boolean | yes | Configured WG UDP port not bound. |
| `compatible` | boolean | yes | Overall pass/fail. `true` only if all blocking checks pass. |
| `warnings` | string[] | yes | Non-blocking issues (e.g., Docker not installed — bootstrap will install). |
| `blocking_failures` | string[] | yes | Blocking issues preventing progression. |

**Compatibility rules**:
- **Pass**: `distro` ∈ {ubuntu, debian} AND `distro_version` ≥ min version (Ubuntu 22.04, Debian 12) AND `disk_free_gb` ≥ 2 AND `has_sudo` == true AND `port_443_free` == true AND `port_wg_free` == true.
- **Warn**: `has_docker` == false (bootstrap installs), `ram_mb` < 1024 (works but tight), `port_80_free` == false (BYO-domain HTTP-01 fails, nip.io HTTP-01 fails, DNS-01 still works).
- **Fail**: unsupported distro, insufficient disk, no sudo, port 443 bound.

**Persistence**: stored as field in `WizardState.preflight_result`. Not a separate file.

### DomainCheckResult

Domain validation result. Generated by `internal/wizard/dnscheck.Validate()`.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `domain` | string | yes | Domain being checked. |
| `mode` | enum | yes | `byo` or `nipio`. |
| `checked_at` | ISO-8601 | yes | When check ran. |
| `a_record_ips` | string[] | yes | Resolved A-record IPs. Empty if no record. |
| `points_to_vps` | boolean | yes | At least one A-record IP matches VPS IP. |
| `cloudflare_detected` | boolean | yes | Nameservers indicate Cloudflare management. |
| `cloudflare_token_valid` | boolean? | no | null if not checked. true/false if Cloudflare detected. |
| `cloudflare_token_scopes` | string[]? | no | Detected scopes if token validated. |
| `cloudflare_zone_id` | string? | no | Zone ID if domain found in CF account. |
| `tls_strategy` | enum | yes | `http-01` or `dns-01`. Determined by Cloudflare detection. |
| `tls_feasible` | boolean | yes | TLS can be provisioned with chosen strategy. |
| `warnings` | string[] | yes | Non-blocking issues (A-record mismatch, slow propagation). |
| `errors` | string[] | yes | Blocking issues (no DNS, token invalid, port 80 blocked for HTTP-01). |

**TLS strategy determination**:
- `byo` + Cloudflare detected + valid token → `dns-01` (wildcard cert)
- `byo` + no Cloudflare → `http-01` (per-subdomain)
- `nipio` → `http-01` (always, nip.io has no DNS-01)

**Persistence**: stored as field in `WizardState.domain_check_result`. Not a separate file.

### InviteLink

Shareable peer-config invite link. Managed by `internal/invite/`.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | UUID (string) | yes | Unique invite identifier. |
| `peer_id` | UUID (string) | yes | Associated peer. FK to 002's Peer entity. |
| `mode` | enum | yes | `hmac_url` or `short_code`. |
| `encrypted_config_blob` | bytes (base64) | yes | AES-256-GCM encrypted WG client config. Key = daemon secret. |
| `token_hash` | string | yes | For HMAC mode: SHA-256 of the random token. For short-code mode: SHA-256 of the code. |
| `hmac_signature` | string? | no | HMAC-SHA256 over `token + peer_id + expires_at`. HMAC mode only. |
| `short_code` | string? | no | 8-digit numeric code. Short-code mode only. Stored as SHA-256 hash. |
| `ttl_seconds` | integer | yes | Time-to-live in seconds from creation. Default: 86400 (24h). Max: 259200 (72h). |
| `created_at` | ISO-8601 | yes | Invite creation timestamp. |
| `expires_at` | ISO-8601 | yes | `created_at + ttl_seconds`. |
| `consumed_at` | ISO-8601? | no | When invite was consumed. Null = not consumed. |
| `consumed_by_ip` | string? | no | IP address of consumer. Null = not consumed. |
| `max_uses` | integer | yes | Maximum consumption count. Default: 1. |
| `use_count` | integer | yes | Current consumption count. Incremented on each valid consume. |
| `created_by_user` | string | yes | "admin" (localhost admin UI) or API token name. |

**Validation rules**:
- `ttl_seconds`: 300 ≤ TTL ≤ 259200 (5 min to 72 hours).
- `max_uses`: 1 ≤ max ≤ 10.
- `short_code`: if present, must be exactly 8 digits (10000000-99999999). Stored as `sha256(code)`, never plaintext.
- `encrypted_config_blob`: AES-256-GCM with 12-byte nonce prepended. Key = first 32 bytes of `sha256(daemon_secret)`.
- One-time consumption (`max_uses == 1`): `use_count` increments to 1 on first consume. Subsequent attempts return `410 invite_consumed`.

**Persistence**:
- File: `~/.unet/invites.jsonl` — append-only JSONL, one JSON object per line.
- Mode: 0600
- GC: on daemon start + every hour, prune entries where `expires_at < now() - 1h` AND `consumed_at != null`.
- Index: in-memory map `token_hash → line_offset` for O(1) lookup. Rebuilt on daemon start by scanning JSONL.

### OnboardingEvent

Structured event emitted during wizard flow. Flows into 005's log pipeline.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `timestamp` | ISO-8601 | yes | Event time. |
| `wizard_session_id` | UUID | yes | Associated wizard session. |
| `event_type` | enum | yes | `step_complete`, `step_error`, `preflight_result`, `domain_check_result`, `commit_start`, `commit_success`, `commit_failure`, `peer_created`, `invite_created`, `invite_consumed` |
| `step` | string? | no | Wizard step name (if applicable). |
| `payload` | object | yes | Event-specific data. |
| `success` | boolean | yes | Whether the event represents a success. |
| `duration_ms` | integer? | no | Duration of the operation (for perf tracking). |

**Emitted via**: `slog.Info("wizard.step_complete", "event", OnboardingEvent{...})` — flows into 005's unified log stream.

**Not persisted separately** — lives in 005's JSONL log files (`~/.unet/logs/daemon-YYYY-MM-DD.jsonl`). Queryable via `GET /v1/logs/stream` with `component=wizard` filter.

### QRConfig

Generated QR code and deeplink data. Ephemeral — not persisted.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `peer_id` | UUID | yes | Associated peer. |
| `config_text` | string | yes | Full WireGuard client config text (including AmneziaWG params). |
| `qr_png_base64` | string | yes | Base64-encoded 256×256 PNG QR code image. |
| `deeplink_uri` | string | yes | `wireguard://import?config=<base64url(config_text)>` |
| `generated_at` | ISO-8601 | yes | Generation timestamp. |

**Not persisted** — generated on demand via `POST /v1/peers/{id}/qr`. Client caches in browser memory. If needed again, re-generate from stored peer config.

## Relationship Diagram

```
WizardState (1) ──── has ──── (0..1) PreflightResult
     │
     └── (0..1) DomainCheckResult

Peer (from 002) ── (0..N) ── InviteLink
     │
     └── (0..1) QRConfig (ephemeral, on-demand)

OnboardingEvent ── references ── WizardState.session_id
                ── references ── Peer.id (on peer_created / invite events)

InviteLinkStore (JSONL file)
     └── contains ── 0..N InviteLink records
```

## Persistence Summary

| Entity | Storage | Lifecycle |
|--------|---------|-----------|
| WizardState | `~/.unet/wizard-state.json` | Created on session start. Deleted on commit. |
| PreflightResult | Embedded in WizardState | Same lifecycle as WizardState. |
| DomainCheckResult | Embedded in WizardState | Same lifecycle as WizardState. |
| InviteLink | `~/.unet/invites.jsonl` (append-only) | Created on invite generation. GC'd after expiry. |
| OnboardingEvent | `~/.unet/logs/daemon-*.jsonl` (via 005) | Retained per 005's retention policy (30d default). |
| QRConfig | In-memory only (not persisted) | Generated on demand, discarded after response. |
